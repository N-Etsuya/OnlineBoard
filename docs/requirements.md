# 要件定義書

## 1. 目的
本プロジェクトは、ローカルネットワーク上で動く、簡易的なオンライン掲示板を構築することを目的とする。ユーザー管理、スレッドと投稿の管理を主な機能とする。

## 2. 機能要件

### 2.1 ユーザー管理
- ユーザー登録: ユーザーはメールアドレス、パスワード、ニックネームを指定して登録できる。パスワードはハッシュ化して保存する。
- ログイン: 登録済みのメールアドレスとパスワードを使用してログインできる。
- ログアウト: ログイン中のユーザーは、通信が終了すると自動的にログアウトされる。

### 2.2 スレッド管理
- スレッド作成: ログイン済みユーザーが新しいスレッドを作成できる。スレッドにはタイトルを設定する。
- スレッド一覧取得: 作成されたスレッドの一覧を取得できる。

### 2.3 投稿管理
- 投稿作成: ログイン済みユーザーが、指定したスレッドに新しい投稿を作成できる。
- スレッド内の投稿一覧取得: 指定したスレッドに属する投稿の一覧を取得できる。

## 3. 非機能要件

### 3.1 セキュリティ
- パスワードのハッシュ化: パスワードは、ハッシュ関数を用いてハッシュ化し、平文では保存しない。

### 3.2 信頼性
- エラー処理: 適切なエラー処理とログ出力を行い、システムの信頼性を確保する。
- データの整合性: データベース操作において、データの整合性を維持する。

### 3.3 拡張性
- モジュール化: 機能を適切にモジュール化し、将来の拡張や変更に対応しやすくする。
- ログ機能: システムの動作をログに記録し、問題の追跡と分析を容易にする。

## 4. システム構成
- プログラミング言語: Java
- データベース: SQLite
- クライアントとの通信: Socket通信
- フロントエンド: JavaFX
